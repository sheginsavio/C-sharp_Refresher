Topic 7: Inheritance and Polymorphism in Real Life
1. Inheritance — Reusing Code
Concept Recap:  
Inheritance lets a child class reuse code from a parent class.

Real-World Instance:  
Imagine you’re building a transportation management system for a company. You have different types of vehicles: cars, trucks, buses, motorcycles. All of them share common features like Brand, Start(), Stop().

Instead of writing these features separately for each vehicle, you create a base class Vehicle and let others inherit from it.

Implementation:

csharp
class Vehicle
{
    public string Brand { get; set; }
    public void Start() => Console.WriteLine($"{Brand} is starting...");
    public void Stop() => Console.WriteLine($"{Brand} is stopping...");
}

class Truck : Vehicle
{
    public int LoadCapacity { get; set; }
    public void LoadCargo() => Console.WriteLine($"{Brand} is loading cargo...");
}

class Bus : Vehicle
{
    public int PassengerCount { get; set; }
    public void PickPassengers() => Console.WriteLine($"{Brand} is picking up {PassengerCount} passengers...");
}
👉 Where used: Logistics companies, ride-sharing apps, fleet management software.
👉 Benefit: Write common code once, reuse everywhere.

2. Polymorphism — Same Method, Different Behavior
Concept Recap:  
Polymorphism allows the same method to behave differently depending on the object type.

Real-World Instance:  
In a payment system, you might have different payment methods: CreditCard, PayPal, BankTransfer. Each has a ProcessPayment() method, but the implementation differs.

Implementation:

csharp
abstract class Payment
{
    public abstract void ProcessPayment(double amount);
}

class CreditCardPayment : Payment
{
    public override void ProcessPayment(double amount)
    {
        Console.WriteLine($"Processing credit card payment of {amount}...");
    }
}

class PayPalPayment : Payment
{
    public override void ProcessPayment(double amount)
    {
        Console.WriteLine($"Processing PayPal payment of {amount}...");
    }
}

class BankTransferPayment : Payment
{
    public override void ProcessPayment(double amount)
    {
        Console.WriteLine($"Processing bank transfer of {amount}...");
    }
}

class Program
{
    static void Main()
    {
        Payment p1 = new CreditCardPayment();
        Payment p2 = new PayPalPayment();
        Payment p3 = new BankTransferPayment();

        p1.ProcessPayment(1000);
        p2.ProcessPayment(500);
        p3.ProcessPayment(2000);
    }
}
👉 Output:

Code
Processing credit card payment of 1000...
Processing PayPal payment of 500...
Processing bank transfer of 2000...
👉 Where used: E-commerce platforms, banking apps, subscription services.
👉 Benefit: You can add new payment methods later without changing existing code — just create a new class that overrides ProcessPayment().

3. Abstract Classes and Interfaces — Enforcing Rules
Concept Recap:  
Abstract classes and interfaces define contracts that child classes must follow.

Real-World Instance:  
In a game development project, you might have different characters (Warrior, Mage, Archer). Each must implement an Attack() method, but the way they attack differs.

Implementation:

csharp
interface ICharacter
{
    void Attack();
}

class Warrior : ICharacter
{
    public void Attack() => Console.WriteLine("Warrior attacks with a sword!");
}

class Mage : ICharacter
{
    public void Attack() => Console.WriteLine("Mage casts a fireball!");
}

class Archer : ICharacter
{
    public void Attack() => Console.WriteLine("Archer shoots an arrow!");
}

class Program
{
    static void Main()
    {
        ICharacter c1 = new Warrior();
        ICharacter c2 = new Mage();
        ICharacter c3 = new Archer();

        c1.Attack();
        c2.Attack();
        c3.Attack();
    }
}
👉 Output:

Code
Warrior attacks with a sword!
Mage casts a fireball!
Archer shoots an arrow!
👉 Where used: Video games, simulations, AI bots.
👉 Benefit: Guarantees that every character has an Attack() method, but allows flexibility in how they attack.

✅ Summary of Practical Use
Inheritance: Used in systems where multiple entities share common behavior (vehicles, employees, products).

Polymorphism: Used in systems where the same action must adapt to different types (payments, notifications, file formats).

Abstract Classes/Interfaces: Used to enforce rules across different implementations (game characters, plugins, APIs).
