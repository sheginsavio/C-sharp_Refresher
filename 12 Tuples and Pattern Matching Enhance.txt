Tuples and Pattern Matching Enhancements
1. Tuples
A tuple is a lightweight way to group multiple values together without creating a class.

Useful when you want to return more than one value from a method.

Example:
csharp
class Program
{
    static (string Name, int Age) GetPerson()
    {
        return ("Alice", 25);
    }

    static void Main()
    {
        var person = GetPerson();
        Console.WriteLine($"Name: {person.Name}, Age: {person.Age}");
    }
}
👉 Output:

Code
Name: Alice, Age: 25
💡 Theory: Tuples are quick and convenient for bundling values together.

2. Pattern Matching Enhancements
Pattern matching lets you check values and types in a more expressive way.

Type Patterns
csharp
object obj = "Hello";

if (obj is string s)
{
    Console.WriteLine($"String value: {s}");
}
👉 Output:

Code
String value: Hello
Relational and Logical Patterns (C# 9+)
csharp
int number = 15;

if (number is > 10 and < 20)
{
    Console.WriteLine("Number is between 10 and 20.");
}
👉 Output:

Code
Number is between 10 and 20.
Switch Expressions with Patterns
csharp
string fruit = "apple";

string color = fruit switch
{
    "apple" => "Red",
    "banana" => "Yellow",
    "grape" => "Purple",
    _ => "Unknown"
};

Console.WriteLine(color);
👉 Output:

Code
Red
💡 Theory: Pattern matching makes decision-making concise and readable.

🌍 Real-World Instances
Instance 1: Returning Multiple Values
Scenario: A function needs to return both a status and a message.

Implementation: Use a tuple (bool Success, string Message).

csharp
static (bool Success, string Message) ProcessOrder(int orderId)
{
    if (orderId > 0)
        return (true, "Order processed successfully.");
    return (false, "Invalid order ID.");
}
👉 Benefit: No need to create a separate class just for two values.

Instance 2: Input Validation
Scenario: Validate user input with pattern matching.

Implementation: Use relational patterns.

csharp
int age = 17;
string result = age switch
{
    < 18 => "Minor",
    >= 18 and < 60 => "Adult",
    _ => "Senior"
};
👉 Benefit: Cleaner logic for categorizing input.

Instance 3: Parsing API Responses
Scenario: An API returns different types of data (string, number, object).

Implementation: Use type patterns.

csharp
object response = 200;

if (response is int code)
    Console.WriteLine($"Status Code: {code}");
else if (response is string msg)
    Console.WriteLine($"Message: {msg}");
👉 Benefit: Safely handle multiple response types.
